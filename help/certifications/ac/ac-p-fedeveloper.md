# Jornada de certificação para Adobe [!DNL Commerce] Profissional de desenvolvedor front-end

Saiba como se tornar um Adobe Developer Professional certificado.

## Experiência mínima

Deve ter entre 0 e 12 meses de experiência prática com Adobe [!DNL Commerce] Desenvolvimento front-end e a capacidade de trabalhar com soluções técnicas para fornecer componentes individuais.

Além disso, você deve exibir conhecimento das seguintes tecnologias:

* Forte compreensão do Adobe [!DNL Commerce] versão 2.4.3
* Noções básicas sobre a estrutura e o layout de front-end
* Experiência e compreensão de HTML, CSS (e pré-processadores), LESS, JavaScript e componentes de interface do usuário, jQuery, Underscore, Require.JS, Knockout, também modelos PHP, conhecimento de BD e gerenciamento de conteúdo
* Familiaridade com a plataforma, Blocos de CMS, Widgets, páginas de CMS, páginas de Categoria, páginas de Produto, Carrinho e Check-out e a área Painel da Conta
* Noções básicas ou conhecimento das configurações do administrador
* Conhecimento sobre a extensão e substituição de estilos, layouts e arquivos JS
* Extensão de temas/hierarquia de temas principais
* comando CLI (limpeza do cache, configuração:atualização, indexadores, modo de implantação) e Grunt
* Tecnologias de armazenamento em cache, navegador do lado do cliente, noções básicas sobre armazenamento (cookies, armazenamento local)
* Noções básicas sobre design responsivo e consultas de mídia
* WYSIWYG/Construtor de páginas
* Noções básicas sobre pacotes de código
* Implementar tradução
* Como entender como trabalhar com modelos de email
* Noções básicas sobre XML
* Noções básicas sobre SEO, otimização e técnicas de desempenho prontas para uso
* Compreensão básica de links canônicos, bem como mesclagem de JavaScript e CSS

>[!NOTE]
>
>Sua certificação é válida por dois anos a partir do momento em que for aprovada. Recomendamos ter a versão mais recente quando possível, pois ela abrange recursos de produto mais recentes.

**Objetivos e âmbito do exame**

Seção 1: Gestão de Temas (Hierarquia de Tema, configuração de imagem, traduções) (14%)

* Descrever Adobe [!DNL Commerce] estrutura de pastas de temas
* Demonstrar a capacidade de criar temas novos ou estender temas existentes
* Demonstrar a capacidade de adicionar frases de tradução personalizadas

Seção 2: XML de layout e modelos (28%)

* Demonstrar a capacidade de trabalhar com instruções XML de layout básico
* Demonstrar a capacidade de usar layouts de página existentes
* Descreva as etapas para estender e substituir o XML
* Demonstrar a capacidade de criar e personalizar modelos
* Descrever a segurança do modelo (saída de saída)

Seção 3: Estilos (24%)

* Descreva a finalidade de styles-m.less, styles-l.less, print.less
* Descreva a diferença entre arquivos parciais específicos (_extensions.less,_extension.less,_theme.less,_widgets.less,_module.less)
* Em um cenário, demonstrar a capacidade de substituir ou estender [!DNL Magento] MENOS
* Descreva os conceitos básicos de MENOS
* Explicar a hierarquia de estilos ( biblioteca, módulos, temas)
* Explicar a compilação do lado do cliente e do lado do servidor

Seção 4: JavaScript (16%)

* Demonstrar o uso básico do gerenciamento de conteúdo
* Descreva as etapas para aplicar/agendar um tema para um escopo específico (site, loja, visualização de loja)
* Descreva as etapas para aplicar alterações de design a categorias, produtos e páginas CMS usando a configuração de administrador
* Descreva etapas para personalizar emails transacionais

Seção 5: Configuração de administrador e criador de página (12%)

* Demonstrar o uso básico do gerenciamento de conteúdo
* Descreva as etapas para aplicar/agendar um tema para um escopo específico (site, loja, visualização de loja)
* Descreva as etapas para aplicar alterações de design a categorias, produtos e páginas CMS usando a configuração de administrador
* Descreva etapas para personalizar emails transacionais

Seção 6: Ferramentas (CLI e Grunt) (6%)

* Descreva o uso de comandos básicos bin/magento
* Descreva o uso de comandos do Composer (instalar, atualizar, exigir, remover)
* Diferenciar o caso de uso apropriado para modos de implantação

**Faça o exame de certificação**

O Adobe usa dois fornecedores para administrar exames de produtos, PSI e Examity.

* **PSI** oferece opções de teste de centro e teste japonês (quando disponível)
* **[!DNL Examity]** oferece uma opção online remota

Para agendar, reprogramar ou cancelar seu exame, basta clicar no fornecedor escolhido e seguir as instruções fornecidas na tela. É tão simples assim!
