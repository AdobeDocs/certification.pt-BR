# Jornada de certificação para o Adobe [!DNL Workfront] Core Developer Expert

{{intro}}

>[!NOTE]
>
>**Novos recursos de estudo disponíveis:** O [Guia de Preparação do Exame](https://app.rockinfo.com/courses/241) fornece um caminho personalizado por meio do processo de preparação do exame de certificação, incluindo os materiais necessários para obter essa certificação. _Se for solicitado a fazer logon no Guia de Preparação do Exame, use sua Adobe ID._

## Descrição do exame {#exam-description}

{{exam-description}}

<br>

### Detalhes do exame {#exam-details}

* Nível: especialista (1-3 anos de experiência)
* Pontuação de aprovação: 35/53
* Tempo: 106 minutos
* Delivery: on-line por encomenda (requer acesso à câmera)
* Idiomas disponíveis: inglês
* Custo: US$ 225 (global) / US$ 150 (Índia)
* Identificação do Exame: AD0-E907

<br>

### Público-alvo {#intended-audience}

* [!DNL Workfront] Administradores do sistema especializados em implementação e configuração de instância nova ou atualizada
* Designer/arquiteto de processos
* Analista de negócios
* Consultor técnico
* Consultor de configuração

<br>

### Experiência mínima {#minimum-experience}

Você deve ter pelo menos 18-24 meses de experiência com o Adobe [!DNL Workfront] e deve ser capaz de:

* Navegue pelo sistema [!DNL Workfront], entendendo onde reside sua funcionalidade
* Configurar todos os aspectos do sistema
* Discutir os requisitos com os clientes e transformá-los em uma solução viável
* Eliminar e modificar o processo existente para melhoria de forma colaborativa
* Entender e aplicar os casos de uso de marketing e PMO
* Recomendar e criar relatórios que deem suporte aos processos do cliente

<br>

### Objetivos e escopo do exame {#exam-objectives}

{{exam-objectives}}

**Seção 1: Administração e configuração do sistema principal (17%)**

* Dado um cenário, descreva maneiras de compartilhar permissões e acesso com eficiência.
* Dado um cenário, identifique quando um status de grupo versus um status de sistema deve ser usado.
* Dado um conjunto de dados, demonstre os recursos de kickstarts.
* Dado um cenário, identifique as práticas recomendadas para usuários de provisionamento automático.
* Dado um cenário, demonstrar como Grupos e Equipes dentro do perfil de um usuário podem afetar o que pode ser visto ou feito em [!DNL Workfront].
* Dado o cenário, recomende uma configuração de placas para atender a um requisito personalizado.
* Dado um cenário, identifique as práticas recomendadas para administração de grupos.

**Seção 2: Entrada, formulários personalizados e iniciação do projeto (13%)**

* Dado um cenário, identifique as áreas de configuração necessárias para dar suporte às alocações de trabalho.
* Dado um cenário, identifique como um valor é passado para um formulário personalizado de projeto.
* Dado o cenário, descreva o que deve ser feito para resolver erros de relatório e cálculo.
* Dado um cenário, identifique elementos de suporte que se prestam a modelos.
* Dado um cenário com vários formulários, determine ações para controlar o acesso, a visibilidade e a dependência.
* Dado um cenário com campos calculados em formulários de vários objetos, identifique a fórmula correta.

**Seção 3: funcionalidade estratégica: gerenciamento de portfólio e programa, gerenciamento de recursos (13%)**

* Dado um cenário, defina ferramentas para capturar alterações no escopo do projeto.
* Dado um cenário, crie uma visualização que identifique em um portfólio se um projeto atende ao nível médio de critérios necessários para ser executado.
* Dado um cenário, descreva maneiras de gerenciar o risco qualitativa ou quantitativamente.
* Dado o cenário, prever um mecanismo para garantir que os projetos com a maior prioridade são criados para serem devidamente financiados.
* Dado um cenário, forneça um mecanismo para garantir que os projetos tenham todo o pessoal necessário usando ferramentas de gerenciamento de recursos.
* Dado um cenário, use as ferramentas de Gerenciamento de recursos para determinar por que um usuário está superalocado.

**Seção 4: Gerenciamento e prova de documentos (13%)**

* Dado um cenário, demonstrar quando utilizar gerenciamento de documentos versus prova
* Dado um cenário, identifique as práticas recomendadas para usar a funcionalidade de visualizador de provas, configurações e marcação
* Dado um cenário, demonstrar como editar e definir permissões em um arquivo
* Dado um cenário, demonstrar como configurar o armazenamento de documentos externo com restrições de segurança
* Dado um cenário, configure uma conexão com o AEM as a Cloud Service ou o AEM Assets Essentials.

**Seção 5: Relatório (11%)**

* Dado um cenário, descreva maneiras de combinar várias colunas de dados em uma única coluna.
* Dado um relatório ou filtro, demonstre os méritos de usar valores curingas.
* Identifique como criar relatórios sobre relacionamentos pai-filho usando relatórios nativos.
* Dado o cenário, descreva como os relatórios facilitam a execução eficiente do trabalho.
* Dado um cenário, ilustre os relatórios do [!DNL Workfront] para monitorar e melhorar a adoção de usuários.
* Dado um cenário, determine como um tipo de campo seria renderizado em um relatório. (Os tipos de campo incluem: rich text, imagens, seleção múltipla)

**Seção 6: Metodologia / práticas recomendadas / casos de uso (22%)**

* Dado um cenário em [!DNL Workfront], demonstre a funcionalidade para configurar o rastreamento de entregas que fazem parte de uma única campanha.
* Dado um cenário em [!DNL Workfront], demonstre a funcionalidade em torno da priorização estratégica e da justificativa do trabalho.
* Dado um cenário em [!DNL Workfront], demonstrar a funcionalidade em finanças, utilização, previsão, taxas/registros de cobrança
* Dado um cenário em [!DNL Workfront], demonstre a funcionalidade em torno da configuração de fluxo de trabalho e aprovações.
* Dado um cenário, recomende uma estrutura de governança em torno da administração do sistema ao expandir a instância de um cliente.
* Dado um cenário, identifique áreas para consideração ao expandir a instância de um cliente para permitir que equipes adicionais trabalhem juntas.
* Dado um cenário usando [!DNL Workfront] Quadros, identifique recursos nativos para mostrar o progresso.
* Dado um cenário, recomende uma estrutura de governança em torno da administração do sistema e de grupos.

**Seção 7: Consultoria comercial (11%)**

* Dado um cenário, descreva o processo necessário para dar suporte total ao esforço de implementação do [!DNL Workfront] em um departamento multifuncional.
* Dado um cenário, explique como adaptar um fluxo de trabalho existente para atender a uma mudança recente em um processo de negócios crítico e em curso.
* Dado um cenário, identifique as configurações mais difíceis de alterar para uma determinada instância.
* Dado um cenário, identifique as principais considerações do gerenciamento de alterações para uma implementação [!DNL Workfront]
* Dado um cenário em que um cliente tem várias equipes, forneça uma recomendação para aumentar a eficiência.
