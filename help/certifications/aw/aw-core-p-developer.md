# Jornada de certificação para Adobe [!DNL Workfront] Core Developer Professional

Saiba como se tornar um profissional certificado da Adobe Developer.

## Experiência mínima

Você deve ter:

* Pelo menos 3 a 6 meses de experiência prática com Adobe [!DNL Workfront]
* Experiência com o Adobe [!DNL Workfront] funcionalidade e conectividade por meio de entrada, aprovação, planejamento, execução e geração de relatórios
* Uma compreensão profunda do Adobe [!DNL Workfront] tipos de objetos, incluindo formulários personalizados e fluxos de trabalho de aprovação, modelos, durações, esforço de trabalho/horas planejadas
* Uma compreensão profunda do Adobe [!DNL Workfront] Configurar, incluindo níveis de acesso, modelos de layout, preferências de projeto e outros métodos para obter a simplicidade do design do usuário final
* Uma compreensão e experiência com o mapeamento de processos
* Compreensão de diferentes Adobe [!DNL Workfront] personas e como participar do suporte

E você deve ter:

* Familiaridade com [!DNL Workfront] API e API Explorer, mas não necessariamente proficiente em chamadas de API
* A capacidade de usar chamadas de API personalizadas para trabalhar com o [!DNL Workfront] e outras APIs
* Familiaridade com o Adobe [!DNL Workfront]
* Familiaridade com conectores nativos e outros Adobe [!DNL Workfront] produtos e serviços
* Familiaridade com o gerenciamento de projetos, incluindo agilidade e cascata, e conceitos de gerenciamento de recursos

>[!NOTE]
>
>Sua certificação é válida por dois anos a partir do momento em que for aprovada. Recomendamos ter a versão mais recente quando possível, pois ela abrange recursos de produto mais recentes.

**Objetivos e âmbito do exame**

Seção 1. Estratégia e consumo (24%)

* Usar portfólios e programas
* Usar filas de solicitação
* Usar formulários personalizados
* Usar aprovações básicas

Seção 2. Orçamento e planeamento (11%)

* Usar o Planejador de Recursos
* Dado um cenário/conjunto de requisitos, aplique as configurações para configurar um template de projeto

Seção 3. Experiência do usuário (25%)

* Ilustrar as práticas recomendadas do gerenciamento de usuários ao criar a experiência do usuário correta
* Em um cenário, aplique a configuração correta para compartilhar e acessar direitos em todo o sistema
* Descreva o que é um Kickstart e reconheça quando usá-lo
* Demonstrar como criar e usar folhas de ponto
* Descreva a finalidade e o valor dos conectores, plug-ins e complementos

Seção 4. Métricas e relatórios (16%)

* Descrever conceitos básicos para criar, enviar e personalizar relatórios
* Explique como criar e personalizar painéis
* Demonstrar como configurar exibições funcionais de relatórios

Seção 5. Execução (24%)

* Demonstrar como configurar fluxos de trabalho de aprovação e prova
* Reconhecer a finalidade de criar um problema para um projeto
* Demonstrar como personalizar status de projeto em nível de grupo
* Identifique os atributos necessários para ativar o Ágil no [!DNL Workfront]
* Identifique as maneiras de inserir o tempo em [!DNL Workfront]
* Identifique os elementos do rastreamento financeiro em [!DNL Workfront]

**Faça o exame de certificação**

O Adobe usa dois fornecedores para administrar exames de produtos, PSI e Examity.

* **PSI** oferece opções de teste de centro e teste japonês (quando disponível)
* **[!DNL Examity]** oferece uma opção online remota

Para agendar, reprogramar ou cancelar seu exame, basta clicar no fornecedor escolhido e seguir as instruções fornecidas na tela. É tão simples assim!
