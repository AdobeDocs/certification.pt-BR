# Jornada de certificação para Adobe [!DNL Workfront Fusion] Developer Professional

Saiba como se tornar um Adobe Developer Professional certificado.

## Experiência mínima

Você deve ter:

* A capacidade de [!DNL Workfront] API e API Explorer, mas não necessariamente proficiente em chamadas de API
* A capacidade de usar chamadas de API personalizadas para trabalhar com o [!DNL Workfront] e outras APIs
* A capacidade de usar módulos para a variável [!DNL Workfront] aplicativo e outras ações como criar, ler, atualizar, excluir e pesquisar
* A capacidade de construir soluções viáveis que produzam resultados comerciais sem atingir [!DNL Fusion] medidas de proteção
* A capacidade de selecionar o módulo de acionador apropriado para obter resultados comerciais
* Uma compreensão de como usar o controle de fluxo (rotas, iteradores/agregadores, fluxo de explicação)
* A capacidade de realizar soluções básicas de problemas e tratamento de erros
* A capacidade de testar e executar com eficácia um cenário
* A capacidade de gerenciar equipes e usuários da organização
* Noções básicas sobre o propósito e o uso de conectores universais para API REST Oauth2

>[!NOTE]
>
>Sua certificação é válida por dois anos a partir do momento em que for aprovada. Recomendamos ter a versão mais recente quando possível, pois ela abrange recursos de produto mais recentes.

**Objetivos e âmbito do exame**

Seção 1: Conceitos técnicos básicos (39%)

* Considerando um [!DNL Fusion] com um formato de campo que precisa estar em outro formato de campo, forneça as possíveis funções que podem ser usadas para transformar os dados corretamente
* Considerando um [!DNL Fusion] que requer que uma função seja aninhada dentro de outra função, selecione a expressão corretamente formada
* Identificar maneiras de usar as equipes para controlar o acesso a conexões e outros [!DNL Fusion] recursos
* Considerando um [!DNL Fusion] identificar limitações do sistema e dos recursos
* Considerando um [!DNL Fusion] que requer manipulação de fuso horário, distinção entre usuário, organização visível e onde é necessário substituir o fuso horário
* Considerando um [!DNL Fusion] selecione o [!DNL Workfront] módulo(s) ou ação(ões)
* Considerando um [!DNL Fusion] selecione a expressão Painel de mapeamento e/ou os módulos que transformariam adequadamente os dados de entrada em dados de saída
* Identifique a(s) maneira(s) correta(s) de utilizar a variável [!DNL Fusion] Ferramenta de desenvolvimento para solucionar erros na execução ou determinar chamadas e respostas feitas a sistemas de terceiros
* Considerando um [!DNL Fusion] em que os dados em um caminho são necessários em outro caminho, forneça uma solução de conjunto para disponibilizar esses dados em caminhos de roteamento adicionais
* Identifique as maneiras de usar ou produzir uma [!DNL Fusion] modelo para uma necessidade comercial
* Considerando um [!DNL Fusion] , identifique o que fazer para exibir execuções e/ou resolver erros
* Identifique as diferentes opções na [!DNL Fusion] configurações de cenário para atender às necessidades do cenário
* Considerando um [!DNL Fusion] , selecione o controlo de fluxo adequado e/ou determine a combinação correta do(s) módulo(s) de controlo de fluxo

Seção 2: Design e arquitetura de cenário (35%)

* Em uma lista de etapas, escolha a sequência correta para analisar JSON e converter em pacotes
* Dado um conjunto de requisitos, identifique o método de pesquisa de dados correto
* Identificar diferenças entre acionadores instantâneos e programados
* Identificar um módulo correto para fazer upload de um documento no [!DNL Workfront]
* Dado um requisito de negócios, identifique a abordagem correta para rastrear que [!DNL Fusion] processou um registro (não um usuário) e quando
* Considerando um [!DNL Fusion] identificar uma oportunidade de simplificar o design e/ou otimizar para manutenção
* Identificar [!DNL Fusion] práticas recomendadas para documentar cenários
* Dada uma situação, identifique uma oportunidade de reduzir o fluxo de dados por meio de um cenário
* Dado um exemplo do número de pacotes mostrados em um módulo de pesquisa e um agregador a seguir, explique as alterações no número de operações processadas e exibidas pelo inspetor de pacote
* Identifique as informações encontradas no inspetor de pacote
* Dada a descrição de uma necessidade de arquivar um cenário por mais de 60 dias e selecionar a única maneira possível (blueprint de importação/exportação)
* Um exemplo de Erro 403 foi fornecido: Proibido, selecione corretamente a origem da mensagem de erro e a causa

Seção 3: Teste e tratamento de erros (16%)

* Identificar os elementos de um plano de teste
* Identificar os elementos de um caso de teste
* Considerando um [!DNL Fusion] , identificar corretamente a diretiva apropriada para lidar com um serviço não confiável
* Identifique o processo para adicionar tratamento de erros personalizado em um [!DNL Fusion] cenário
* Identifique como rastrear, ler e resolver execuções incompletas
* Considerando um [!DNL Fusion] com dados necessários ausentes, selecione maneiras de lidar com dados inválidos

Seção 4: Trabalho com APIs (10%)

* Dada uma API de terceiros que retorna um 429: Demasiadas solicitações, identifique uma solução para evitar ou lidar com o erro
* Considerando um [!DNL Fusion] em que a nova funcionalidade não está disponível no [!DNL Workfront] , identifique a referência correta e o tipo de módulo para usar a nova funcionalidade
* Dado um sistema de terceiros que não tenha um aplicativo dedicado, identifique o aplicativo HTTP e selecione o módulo apropriado
* Identificar a diferença entre operações CRUD e outros recursos comuns de REST APIs

**Faça o exame de certificação**

O Adobe usa dois fornecedores para administrar exames de produtos, PSI e Examity.

* **PSI** oferece opções de teste de centro e teste japonês (quando disponível)
* **[!DNL Examity]** oferece uma opção online remota

Para agendar, reprogramar ou cancelar seu exame, basta clicar no fornecedor escolhido e seguir as instruções fornecidas na tela. É tão simples assim!
