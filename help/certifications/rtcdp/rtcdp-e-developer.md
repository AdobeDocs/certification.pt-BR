{{exam-description}}

### Detalhes do exame {#exam-details}

* Nível: especialista (1-3 anos de experiência)
* Pontuação de aprovação: 45/68
* Tempo: 136 minutos
* Delivery: on-line por encomenda (requer acesso à câmera)
* Idioma: inglês
* Custo: US$ 225 (global) / US$ 150 (Índia)
* Identificação do Exame: AD0-E605

<br>

### Público-alvo {#intended-audience}

* Desenvolvedor de software
* Engenheiro de dados
* Arquiteto de dados
* Arquiteto de soluções
* Arquiteto corporativo

<br>

### Experiência mínima {#minimum-experience}

Você deve ter:

* Noções básicas sobre diferenças de avaliação de segmento (fluxo, lote, borda)
* Noções básicas sobre casos de uso para APIs (como criar um projeto do desenvolvedor, como a autenticação funciona)
* Noções básicas do serviço de identidade
* Compreensão do fluxo de dados
* Noções básicas do Adobe I/O
* Compreensão de como as relações funcionam
* Capacidade de criar um modelo de dados (esquema, identidade, relacionamentos) para caso de uso do cliente
* Capacidade de criar estratégias para mesclar os dados com base na política de mesclagem
* Noções básicas sobre como determinar a maneira correta de rotular dados, aplicar políticas e Privacy Service
* Familiaridade com limites do padrão de assimilação de dados
* Entender destinos e quais usar
* Noções básicas sobre a estrutura de coleta de dados de borda
* Capacidade de determinar quais Permissões da AEP precisam ser configuradas (ou seja, permissões no nível do atributo) (fazer recomendações aos clientes)
* Noções básicas sobre casos de uso do serviço de consulta
* Compreensão básica do SDK de fontes/destinos

Familiarize-se com estes conceitos:

* Diagramas relacionais de entidade
* Administrador/permissões
* APIs da Experience Platform
* Adobe Experience Platform Debugger
* SDKs da Web e móveis

Você deve se sentir confortável nesses ambientes:

* AEP
* Rede Edge

<br>

### Objetivos e escopo do exame {#exam-objectives}

{{exam-objectives}}

**Arquitetura de dados (19%)**

* Com base em um cenário, converta o RDBMS para o modelo de dados NoSQL do Adobe [!DNL RT-CDP]
* Demonstrar práticas recomendadas para modelagem do Perfil do cliente em tempo real
* Projetar a estratégia de identidade e as relações para modelagem

**Perfil de cliente em tempo real (15%)**

* Descrever como funciona o assembly de perfil
* Descrever como o Gráfico de identidade funciona
* Descrever a composição do perfil
* Diferenciar entre borda e perfil de hub

**Assimilação de dados (13%)**

* Descrever os recursos de assimilação de dados com a CDP
* Descrever os recursos da assimilação do Edge
* Descrever as técnicas avançadas em torno da arquitetura de assimilação de dados

**Segmentação (18%)**

* Descrever diferentes maneiras de criar públicos e segmentos na CDP
* Explicar os vários tipos de segmentação e como eles operam
* Demonstrar uma compreensão de como aplicar casos de uso à ativação de segmento/público-alvo

**Ativação (12%)**

* Explicar os diferentes padrões de ativação na CDP
* Explicar as medidas de proteção ao redor da ativação
* Descrever na personalização do site

**Governança (9%)**

* Aplicar práticas recomendadas para governança na implementação do Adobe [!DNL RT-CDP]
* Descrever como gerenciar dados nos vários serviços

**Administração (15%)**

* Entender os recursos de monitoramento
* Descrever alertas
* Descrever o uso de licença diferente
* Descreva casos de uso para habilitação de ABAC no Adobe [!DNL RT-CDP]
